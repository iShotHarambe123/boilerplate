Arbetsprocess

Så här sattes boilerplaten upp, hur automatiseringen funkar och hur publiceringen är tänkt att ske.

Mål & krav

Skapa en enkel boilerplate för webbutveckling med HTML, CSS och JavaScript som: automatiserar minifiering/transpilering, optimerar bilder, har tydlig filstruktur (src/ → dist/), och kan auto-publiceras via Git när koden mergas till huvud-branch.

Filstruktur

src/
  index.html
  images.html
  process.html
  css/style.css
  js/script.js
  images/ (originalbilder)

Verktyg

  • Node.js + npm för scripts och paket.
  • Parcel som bundler (dev-server, minifiering, transpilering, cache-busting).
  • Sharp för att generera .avif , .webp samt .png från originalbilder (JPG/PNG/SVG).
  • Git/GitHub för versionshantering och deployment-trigger.
  • Netlify för hosting av webbplatsen.

Automatisering (scripts)

Kör lokalt under utveckling och vid build/publicering:

npm run dev     # startar utvecklingsserver (Parcel)
npm run build   # bygger optimerat till dist/
npm run images  # genererar .avif och .webp från src/images/*.jpg|png
npm run favicon  # genererar .png från src/images/*.svg

Bildoptimering

Originalbilder läggs i src/images/. Ett Node-script (Sharp) skapar .avif och .webp. I HTML används <picture> med AVIF och WebP samt JPG/PNG-fallback. 

<picture>
  <source srcset="./images/img1.avif" type="image/avif">
  <source srcset="./images/img1.webp" type="image/webp">
  <img src="./images/img1.jpg" alt="Bild 1" loading="lazy">
</picture>

Faviconen skapas från en favicon.svg och konverteras till favicon.png med convert-favicon.js via npm run favicon. En .ico-version görs manuellt för äldre webbläsare. Alla tre formaten länkas i <head> på varje sida.

Branching & commits

  • Dagligt arbete i dev. När en del är klar → merge till main för publicering.
  • Commit-stil: relativt små, logiska commits med t.ex. feat/fix/chore/style.

Publicering (plan)

Koppla main-branchen till en host (Netlify). Vid push/merge till main: kör npm ci, npm run images (om man vill generera på servern) och npm run build, publicera sedan innehållet i dist/.

Hur man kör projektet

npm install
npm run images   # valfritt under dev, men krävs om man vill ha AVIF/WebP lokalt
npm run dev      # http://localhost:1234
# För release:
npm run build

Lärdomar / Motivering

  • Parcel ger minifiering och transpilering “out of the box” på build, mindre konfig att underhålla.
  • Sharp-scriptet gör bildflödet snabbt, jag lägger bara in en originalbild.
  • Simpel och minimal HTML/CSS/JS gör boilerplaten lätt att återanvända i nya projekt.

Checklista – verifiering

  • Build körd: npm run build (prod-läge).
  • HTML/CSS/JS minifierade i dist/ (JS bundlat med hashade filnamn).
  • Transpilering aktiv via Parcel + browserslist (bakåtkompabilitet).
  • Bildoptimering: <picture> med AVIF/WebP + fallback fungerar.
  • Favicons på plats: favicon.svg, favicon.png, favicon.ico länkade i <head>.
  • Responsivitet testad (mobil/desktop), kodblock <pre>/<code> wrappar korrekt.
  • Navigation: .active-klass visar aktuell sida.
  • Git-flöde: arbete i dev, merge till main för release.
  • Deploy: Netlify byggde från main med npm ci + npm run build, publicerar dist/.
